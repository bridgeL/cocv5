# 需求文档：Tool气泡复用

## 需求背景

当前前端在处理 MCP 工具调用时，当 `tool_start` 消息到来会生成工具调用参数的气泡，当 `tool_end` 消息到来时会**新建一个气泡**来显示工具执行结果。这导致同一个工具的调用参数和结果显示在两个不同的气泡中，用户体验不够连贯。

## 需求目标

将 tool 的结果直接填充到对应的 tool 气泡里，而不是新建一个气泡。

## 具体需求

### 1. tool_start 阶段
- 为每个工具气泡添加唯一标识（`tool_call_id`），用于后续关联
- 在气泡中预留结果展示区域，初始状态隐藏

### 2. tool_end 阶段
- 根据 `tool_call_id` 找到对应的工具气泡
- 在原有气泡中显示结果区域
- 更新状态标识（"调用中" → "成功/失败"）
- 将结果数据填充到结果区域
- 不再新建气泡

## 涉及消息格式

WebSocket 消息类型（已在 `docs/ws/agent.md` 中定义）：
- `tool_start`: 包含 `tool_calls` 数组，每个元素有 `id`, `function` 等字段
- `tool_end`: 包含 `results` 数组，每个元素有 `tool_call_id`, `status`, `result` 等字段

## 预期效果

用户看到的是一个完整的工具调用卡片，包含：
- 工具名称
- 调用状态（从"调用中"动态更新为"成功"或"失败"）
- 参数信息
- 返回结果（tool_end 后填充显示）
